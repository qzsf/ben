# SimPy Visitor Simulation

This is a SimPy model of a Social Security field office where visitors wait for either:

- a priority-based clerk,
- a first-come-first-served kiosk,
- or a patience timeout.

The office is open for 420 simulated minutes. Appointments are generated by slot, walk-ins follow a time-varying random arrival rate, and appointment no-shows are included.
Each visitor gets one of these case types:

- `Card_Replacement`
- `Retirement_Benefits`
- `Disability_Claim`

Each case type has its own average service time, and appointments and walk-ins use different case-type probability mixes.
When a clerk and kiosk are both available at the same time, the visitor randomly uses one of them.
SimPy handles the service limits: only 10 clerk services and 2 kiosk services can happen at the same time.
Detailed event logging is off by default. Set `PRINT_EVENTS = True` in `simulation.py` to print every arrival and service event.
The simulation writes detailed visitor records to `visitors.json`.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Run

```bash
python simulation.py
```

Lower priority numbers are served first by clerks. For example, priority `0` is higher priority than priority `1`.

## Staffing Analysis

Use `staffing_analysis.py` to search for the minimum clerk count needed to meet wait-time goals across repeated simulated days:

```bash
python staffing_analysis.py
```

Example with custom goals:

```bash
python staffing_analysis.py --replications 200 --max-clerks 20 --appointment-p90-wait 15 --walkin-p90-wait 45
```

The default goals are:

- appointment average wait <= 10 minutes
- appointment 90th percentile wait <= 20 minutes
- walk-in average wait <= 30 minutes
- walk-in 90th percentile wait <= 60 minutes
- overall 90th percentile wait <= 45 minutes
- walk-in abandonment rate <= 10%

## Schedule API

The FastAPI schedule endpoints use SQL Server by default with a trusted connection:

```text
DRIVER={SQL Server};
SERVER=SQL-01;
DATABASE=TestDB;
Trusted_Connection=yes;
```

Run the API with:

```bash
.venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

You can override the defaults with `DATABASE_URL` or with `SQLSERVER_HOST`, `SQLSERVER_DATABASE`, `SQLSERVER_DRIVER`, and `SQLSERVER_TRUSTED_CONNECTION`.
